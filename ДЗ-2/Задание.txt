1.1. Для справочника "Контрагенты" необходимо создать нетиповую форму объекта.
"1.2. Поля ""Фамилия"", ""Имя"" и ""Отчетство"" необходимо поместить в одну группу ""ГруппаФИО"".
Поля ""ИНН"" и ""КПП"" необходимо поместить в другую группу ""ГруппаРеквизиты""."
"1.3. На этой форме необходимо реализовать обработку изменения реквизита ""Тип контрагента"".
 
При установке реквизита в значение ""Физ. лицо"" должны отображаться поля ""Имя"", ""Фамилия"", ""Отчество"".
При указании значения ""Юр. лицо"" на форме должны отображаться поля ""ИНН"", ""КПП"" (а предыдущие поля должны наоборот скрываться).
Если значение реквизита не установлено (Неопределено), то указанные поля должны быть скрыты."
"1.4. Видимость поля можно устанавливать следующим образом:
 
Элементы.<ИмяЭлементаФормы>.Видимость = Истина/Ложь;
 
Обновление видимости полей формы должно происходить в отдельной клиентской процедуре. Эта процедура должна вызываться при изменении реквизита ""Тип контрагента"" и при открытии формы. Открытие формы можно обработать в отдельном обработчике ""ПриОткрытии"", который есть в свойствах самой формы.
 
Проверку реквизита можно сделать примерно следующим образом:
 
Если Объект.ТипКонтрагента = ПредопределенноеЗначение(""Перечисление.ТипыКонтрагентов.ФизЛицо"") Тогда

ИначеЕсли Объект.ТипКонтрагента = ПредопределенноеЗначение(""Перечисление.ТипыКонтрагентов.ЮрЛицо"") Тогда
 
// --- устанавливаем видимость других полей
 
Иначе
 
// --- скрываем все поля
 
КонецЕсли;"
Необходимо создать общую форму "Конфигурация->Общее->Общие формы". Форма должна представлять собой калькулятор и содержать кнопки (0,1,2,3,4 ... 9), кнопки мат. операций (+, - , /, *), кнопку для сброса расчета "C" и кнопку "=". Форма должна содержать поле, в котором выводится набранное значение и результат расчета, как в стандартном калькуляторе. Для хранения промежуточных вычислений можно использовать реквизиты формы, которые не отображаются в интерфейсе.
